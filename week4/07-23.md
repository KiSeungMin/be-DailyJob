# be-DailyJob

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다.
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

<br>

###### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습: 원래 오늘은 ERD와 API 명세서 작업을 진행하기로 계획했습니다. 둘 다 시간이 꽤 걸릴 것 같아서 오늘 안에 완성까지는 생각하지 못했는데, 작업 속도가 예상보다 빨라서 오늘 안에 둘 다 끝낼 수 있었습니다.
- 아직 끝내지 못한 부분: 와이어프레임에서 아직 미구현된 화면이 많이 남아있습니다. 이 부분은 오늘 집에 가서 마저 구현할 예정입니다.

<br>

###### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: ERD Cloud에 MCP를 연동해서 ERD를 편집하기로 한 게 오늘 가장 의미있는 판단이었습니다. 팀원들과 노션에 정리해둔 기획안과 기능 명세서를 근거로 ERD 틀을 잡는 방식으로 진행했습니다.
- 그렇게 판단한 이유: ERD Cloud에서 컬럼이나 값을 하나하나 입력하는 데 시간이 꽤 오래 걸리는데, 이미 노션에 기획안과 기능 명세서가 정리돼 있으니 그걸 근거로 틀을 먼저 잡으면 손으로 반복 입력하는 시간을 크게 줄일 수 있겠다고 봤습니다.
- 고려했지만 선택하지 않은 대안: ERD Cloud에서 처음부터 전부 수기로 입력하는 방법도 있었지만, 그러면 오늘 안에 ERD와 API 명세서를 둘 다 끝내는 건 어려웠을 겁니다.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 명세서를 근거로 틀을 잡다 보니 ERD가 기획/명세와 어긋나지 않게 시작할 수 있었고, 반복 입력에 쓸 시간을 테이블 관계나 컬럼 정합성을 검토하는 데 더 쓸 수 있었습니다. 결과적으로 예상보다 훨씬 빠르게 ERD를 완성했습니다.

<br>

###### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 화면 구성이 기능 명세서와 일치하지 않거나, 아직 팀원끼리 생각이 통일되지 않은 부분이 많았습니다.
- 처음 추측한 원인: 문서는 이미 정리해뒀으니 화면도 그에 맞게 다 반영돼 있을 거라고 생각했습니다.
- 원인을 확인하기 위해 시도한 방법: 팀원들과 다 같이 기능 명세서와 와이어프레임을 하나씩 대조하면서, 서로 생각이 다른 부분과 문서에는 있는데 화면에 반영이 안 된 부분을 찾아나갔습니다.
- 실제 원인: 문서를 화면으로 옮기는 과정에서 누락된 화면이 있었고, 일부는 애초에 팀원끼리 합의가 덜 된 채로 넘어간 부분이었습니다.
- 해결 방법: 다 같이 대조하면서 찾다 보니 서로 생각도 자연스럽게 맞춰지고, 누락된 화면도 어렵지 않게 찾아낼 수 있었습니다.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 문서를 화면으로 옮긴 뒤에는 "옮겼으니 됐다"고 넘기지 말고, 명세서와 와이어프레임을 항목 단위로 한 번 대조하는 단계를 넣기로 했습니다.

<br>

###### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: ERD 작업처럼 반복 입력이 많은 단계에서, 그냥 손으로 다 하지 않고 이미 정리된 문서를 근거로 도구를 붙여 속도를 높이는 쪽을 빠르게 정하고 실행했습니다.
- 그 접근이 도움이 된 이유: 오래 걸릴 거라고 예상했던 작업을 하루 안에 끝내면서, 남는 시간을 정합성 검토나 다른 작업에 돌릴 수 있었습니다. 도구를 쓰되 근거 문서를 바탕으로 틀만 잡고 세부는 직접 확인하는 선을 지킨 것도 결과가 명세와 어긋나지 않게 하는 데 도움이 됐습니다.
- 다음에도 반복하고 싶은 점: 반복 작업이 많은 단계에서는 손으로 끝까지 밀어붙이기 전에, 이미 있는 자료를 근거로 속도를 높일 방법이 있는지 먼저 판단해보는 습관을 유지하고 싶습니다.

<br>

###### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점: 하루 막판에 조금 지쳐서, 회의할 때 팀원들이 보기에 제가 피곤해 보였을 수도 있을 것 같습니다.
- 그렇게 된 원인: 오늘 작업량을 빠르게 밀어붙이다 보니 후반부에 체력이 떨어졌고, 그게 회의 태도에서 티가 났을 수 있습니다.
- 다음에는 다르게 해볼 구체적인 행동: 팀장으로서 제 컨디션이 회의 분위기에 영향을 줄 수 있으니, 중간중간 페이스를 조절하고 회의에는 지장이 가지 않도록 신경 쓰기로 하겠습니다.
